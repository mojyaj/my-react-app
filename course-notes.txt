ReactJs
    - ReactJs is a library & NOT a Framework
    - Used to easily build and arrange interfaces
    - Uses components
        - a self-contained section of code that functions as 
            a reasonable building block
    - ReactJs uses a JavaScript syntax extension called, JSX
        - JSX means JavaScript XML
        - Allows to write HTML like code within JavaScript files
    - Uses VirtualDom
        - Lightweight version of a real DOM of a web packages
        - Make changes to VirtualDom and only apply specific changes to
            the real DOM
        - reduces rendering performance overhead
    - Prerequisites
        - arrays
        - classes
        - objects
        - arrow functions
        - HTML
    - Project requirements
        - Node Package Manager (npm)
    - vite is a development server
        - Build Tool (W3Schools)
        - install command: npm install -g create-vite
        - modern replacement to OUTDATED Create-React-App

Style React Components with CSS

3 CSS Styling methods
1) External CSS
2) Modules CSS  *Purposed for React Components
3) Inline CSS

1. External
    - Easy for simple projects
    - Flexibility with Media Queries & Psuedo Selectors
        - @media only screen and (max-width: 600px) { ... }
        - .button:focus
    - 'index.css'
    - Can lead to naming conflicts while working with many components/Modules
2. Modules
    - Dedicated Style Sheet for each component
    - Scoped
    - React Modules naming convention
        - 'component.module.css'
    - Styles are applied to components via the 'className' attribute
        - example, <button className="styles"> </button>
    - Avoids Naming Conflicts
        - Each className is unique
        - Generated class name via hashing algorithm
    - Each Component has their very own styles
    - Lacks global CSS setup like normal external CSS files
    - GLOBAL CSS + CSS Modules
        - import at top level in 'main.jsx'
        - 'globals.css' or 'index.css'
        - shared utility classes (.container, .btn)
3. Inline
    - Create an object to store styles
    - Great for components with minimal styling
    - Convenient
        - No classes/ids
        - No Naming Conflicts with global styles
    - Downsides
        - Less maintainable in larget applications
        - Not great for complex styling
        - Reduces readability of components

3 Topics
    Props
    prop-types
    Default props

Props
    Read-Only (immutable) properties that are shared between componnents.
    A parent componenet can send data to a child component.
    <Component key=value />

    'Props' are arguments passed into React components via HTML attributes
     Example, 
        Component:
        <Student name="Spongebob" age={30}  isStudent={false}> </Student>
                 ^^^^             ^^^       ^^^^^^^^^

        *Values that are type String should be encased in double-quotes ("").

PropTypes
    Validate props at runtime in React apps
    A mechanism that ensures that the passed value is of the correct datatype.
    Only adds WARNINGS in development so you catch bugs early.
    Only works in Development. No effect in production.
    Modern React developers use TypeScript, NOT prop-types.
    npm install prop-types
    Module,
        import PropTypes from 'prop-types'
    Example,
        Declare the 'propTypes' of the 'prop' called, 'Student'
            Student.propTypes = {
                name: PropTypes.string,
                age: PropTypes.number,
                isStudent: PropTypes.bool
            }
        ** See Default Props section below
            it creates a prop differently

    Common PropTypes
        User.propTypes = {
            name: PropTypes.string.isRequired, // This prop is REQUIRED
            age: PropTypes.number,
            isAdmin: PropTypes.bool,
            hobbies: PropTypes.array,
            user: PropTypes.object,
            onClick: PropTypes.func,
        };     

    Default Props
        Default values for props in case they are not passed from the parent component
        defaultProps does not reliably work with modern function components anymore in React.
            Officially removed in React 19
        defaultProps DO WORK for class components
        *TypeScript is still prefered over defaultProps
        Default values only apply when the prop is "undefined" NOT when its "null"
    Example,
        Component.defaultProps = {
            name: "John",
            age: 0,
        }

ES6/JavaScript Default Parameters
    Modern Alternative to defaultProps
    Use ES6 Default Parameters directly in the function signature instead

useState() Hook

React Hook
    Special function that allows functional components to use React features
    without writing class components (Reactv16.8).
    React Hook functions let you “hook into” React features inside function components.

useState()
    A React hook that allows the creation of a stateful variable AND a setter
    function to update its value in the Virtual DOM.
    [name, setName] = useState()
    - Adds state (memory) to a component

React Hook Module
    - import React, { useState } from 'react'
    - 'React' is default export
    - useState, useEffect, useContext, useReducer, useCallback, and more
    - Encase stateful a variable and its setter function inside square 
        brackets [] (Array descructoring)
    - state can be anything - number, string, boolean, array, object
    - Never modify stateful 'variables' directly
        - Only use the 'setter'
    - Never manipulate the DOM directly in React components
        - Use React Hooks to manipulate the DOM instead
    - Only call hooks at the top level
    - Only call hooks inside React functions
    - State updates are asynchronous
        Calling this sequentially does not work:
            setCount(count + 1)
        but calling this sequentially works:
            setCount(prev => prev + 1)
